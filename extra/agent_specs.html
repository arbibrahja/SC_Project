Planner / Orchestrator

The "brain" of the system. Receives natural language queries, calls the Claude API to produce a JSON execution plan, routes work to appropriate agents in sequence, aggregates results, and generates a narrative response with follow-up suggestions.

Responsibilities
Query Understanding
Parses natural language, identifies intent and OLAP operation type
Agent Selection
Maps query type to the correct agent(s)
Coordination
Passes context between agents, aggregates outputs
Context Management
6-turn sliding window conversation history for follow-ups
Fallback Planning
Rule-based keyword planner when Claude API is unavailable

DimensionNavigator Agent 1 ‚Äî Required

Navigates dimensional hierarchies across Time (Year‚ÜíQuarter‚ÜíMonth), Geography (Region‚ÜíCountry), and Product (Category‚ÜíSubcategory). Enables progressive exploration from summary to detail and back.

Operations
drill_down
Navigate from a higher level to a more detailed level within a hierarchy
roll_up
Aggregate from detail to a higher summary level
group
Group-by on any combination of dimensions with full measures
Input Schema (AgentInput)
FieldTypeRequiredDescription
operationstring‚ú±drill_down | roll_up | group
parameters.hierarchystring"time" | "geography" | "product" (default: "time")
parameters.to_levelstringTarget level: "quarter", "country", "subcategory"
parameters.filtersobjecte.g. {"year": 2024, "region": "Europe"}
parameters.dimensionsarrayFor group operation: ["region", "category"]
contextstringOriginal user query for narrative generation
Output Fields (AgentOutput)
FieldDescription
dataDataFrame with dimension columns + total_revenue, total_profit, avg_margin, total_qty, transactions
summarye.g. "Drill-down on time to 'quarter' level. 4 rows. Top: Q4 with $1,784,320 revenue."
metadata.hierarchyHierarchy navigated
metadata.levelResulting level reached
Example Natural Language Triggers
üí¨"Break down 2024 revenue by quarter" ‚Üí drill_down time to quarter, filter year=2024
üí¨"Show quarterly totals as a yearly summary" ‚Üí roll_up time to year
üí¨"Drill into Europe by country" ‚Üí drill_down geography to country, filter region=Europe
üí¨"Revenue by region and category" ‚Üí group dimensions=[region, category]

CubeOperations Agent 2 ‚Äî Required

Performs classical OLAP cube manipulations. Slice applies a single-dimension filter, Dice applies multi-dimension filters, Pivot reshapes dimensions into columns, and Drill-Through exposes the raw underlying transactions.

Operations
slice
Fix one dimension value, group by another ‚Äî e.g. "only 2024"
dice
Filter multiple dimensions simultaneously ‚Äî e.g. "Electronics in Europe"
pivot
Rotate view: row_dim √ó col_dim matrix of a measure
drill_through
Return actual transaction records (not aggregated)
Input Schema (AgentInput)
OperationKey Parameters
slicefilter: {"year": 2024}, group_by: ["region"]
dicefilters: {"year": 2024, "region": "Europe", "category": "Electronics"}, group_by: ["country"]
pivotrow_dim: "region", col_dim: "year", measure: "revenue", filters: {}
drill_throughfilters: {...}, limit: 50 (default)
Supported Dimension Filter Keys
yearquartermonth regioncountry categorysubcategory customer_segment
Example Natural Language Triggers
üí¨"Show only 2024 sales by region" ‚Üí slice filter={year:2024} group_by=[region]
üí¨"Electronics in Europe" ‚Üí dice filters={category:Electronics, region:Europe}
üí¨"Revenue matrix: regions vs years" ‚Üí pivot row_dim=region, col_dim=year, measure=revenue
üí¨"Show me the actual transactions from Q4" ‚Üí drill_through filters={quarter:Q4}

KPICalculator Agent 3 ‚Äî Required

Computes business KPIs including year-over-year growth, month-over-month change, period comparisons, profit margin analysis, and Top-N rankings. The most frequently used agent for business intelligence queries.

Operations

Year-over-year revenue growth % by any dimension
mom_change
Month-over-month revenue change for a given year

Side-by-side comparison of two time periods
top_n

profit_margins
Margin analysis: avg and blended margin by dimension

Overall KPI dashboard: revenue, profit, margin, AOV

AnomalyDetection Agent 6 ‚Äî Optional

Applies statistical methods to identify unusual patterns in sales data. Uses Z-score for time-series anomalies (flagging months with revenue 2+ standard deviations from the mean) and IQR for product margin outliers.

Operations & Statistical Methods
monthly_anomaly
Z-score method. Flags months where |Z| > 2.0. Computes mean and std across all months.
product_anomaly
IQR method. Flags subcategories outside Q1 ‚àí 1.5√óIQR or Q3 + 1.5√óIQR margin range.
Output Columns ‚Äî monthly_anomaly
ColumnTypeDescription
year, month, month_nameint/strTime dimensions
monthly_revenuefloatTotal revenue for the month
z_scorefloatStandard deviations from mean. |z|>2 = anomaly
anomalyboolTrue if flagged as anomalous
anomaly_typestring"High" | "Low" | "Normal"

üí¨"Detect revenue anomalies" ‚Üí monthly_anomaly
üí¨"Are there any unusual months?" ‚Üí monthly_anomaly
üí¨"Which products have abnormal margins?" ‚Üí product_anomaly
üí¨"Find outliers in profit margins" ‚Üí product_anomaly
